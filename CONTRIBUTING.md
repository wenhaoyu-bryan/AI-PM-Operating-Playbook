# Contributing to AI PM Operating Playbook

Thanks for your interest in contributing! This playbook is a living resource for AI Product Managers.

## How to contribute

### Submit a case study

Used the AI PM Canvas, PRD-lite, or Case Study Template in your own project? We'd love to hear about it.

1. Open a [Case Study issue](../../issues/new?template=case-study.yml)
2. Fill in the template with your project details
3. We'll review and help format it for the playbook

### Suggest improvements

Found something unclear, missing, or broken?

1. Open a [Feedback issue](../../issues/new?template=feedback.yml)
2. Describe what you'd like to see changed

### Submit a PR

For direct content contributions:

1. Fork the repo
2. Create a feature branch: `git checkout -b feature/your-improvement`
3. Make your changes
4. Ensure bilingual content (English + Chinese) for new material
5. Submit a PR using the PR template

## Content guidelines

- **Bilingual**: All new content should have both English (`*.md`) and Chinese (`*.zh-CN.md`) versions
- **Practical**: Focus on frameworks and workflows, not theory
- **No confidential info**: Generalize case details to avoid exposing proprietary business context
- **Link checks**: Make sure all internal links work

## Tech setup (for site development)

```bash
npm install
npm run dev       # Start dev server
npm run build     # Build for production
npm run preview   # Preview production build
```

## Code of conduct

Be respectful, constructive, and focused on making this resource better for the AI PM community.
