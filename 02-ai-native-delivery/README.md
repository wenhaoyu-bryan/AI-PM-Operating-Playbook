# AI-Native Delivery

This section covers how an AI PM uses AI-assisted development workflows to move from product idea to working prototype.

It includes notes on:
- Vibe Coding — using coding agents to turn product intent into prototypes
- Harness engineering — directing AI to write code
- PM-led prototyping — structured workflows for rapid prototype development
- Requirements breakdown — turning vague requirements into structured tasks
- Technical stacks — proven frontend stacks for rapid prototyping

- [Harness Engineering](harness-engineering.md) — The practice of directing AI to write code, not writing it yourself

## When to use this

- When you need to go from product idea to working prototype quickly
- When structuring a collaboration with an AI coding tool
- When breaking down vague requirements into actionable tasks

## Connection to AI PM work

AI-native delivery is what separates an AI PM who writes documents from one who ships prototypes. These workflows are not about replacing engineers — they are about enabling PMs to validate ideas faster, communicate more concretely, and reduce the gap between concept and demo.
