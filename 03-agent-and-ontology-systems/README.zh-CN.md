# Agent 和本体系统

本版块涵盖知识驱动 AI 系统的产品方法论——本体设计、Agent 工作流模式和 Human-in-the-loop 架构。

**[English](README.md)** | **中文**

## 适用场景

- 设计需要结构化知识表示的产品
- 规划带工具使用和推理步骤的 Agent 工作流
- 定义人工审核在 AI 驱动流程中的位置
- 评估本体 vs. Schema vs. 数据模型的决策

## 核心概念

**本体驱动的产品设计：** 围绕业务实际如何看待其领域来构建产品——实体、关系、约束——而不是围绕数据库如何存储数据。

**Agent 工作流设计：** 规划 AI Agent 遵循的推理、工具使用和决策点序列，包括它应该在哪里停下来寻求人工输入。

**知识架构：** 理解原始数据、结构化 Schema、语义本体和驱动智能产品的知识图谱之间的层次。

## 与 AI PM 工作的连接

基于知识图谱和 Agent 工作流构建的产品，需要与传统 CRUD 应用不同的设计方法。AI PM 需要理解数据如何变成知识、知识如何变成决策、以及 AI 自主性的边界应该设在哪里。
