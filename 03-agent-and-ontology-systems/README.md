# Agent and Ontology Systems

This section covers product methodology for knowledge-driven AI systems — ontology design, agent workflow patterns, and human-in-the-loop architecture.

**English** | **[中文](README.zh-CN.md)**

## When to use this

- When designing products that need structured knowledge representations
- When planning agent workflows with tool use and reasoning steps
- When defining where human review fits in an AI-powered process
- When evaluating ontology vs. schema vs. data model decisions

## Key concepts

**Ontology-driven product design:** Structuring the product around how the business actually thinks about its domain — entities, relationships, constraints — rather than around how the database stores data.

**Agent workflow design:** Planning the sequence of reasoning, tool use, and decision points an AI agent follows, including where it should stop and ask for human input.

**Knowledge architecture:** Understanding the layers between raw data, structured schemas, semantic ontologies, and the knowledge graphs that power intelligent products.

## Connection to AI PM work

Products built on knowledge graphs and agent workflows require a different design approach than traditional CRUD applications. The AI PM needs to understand how data becomes knowledge, how knowledge becomes decisions, and where the boundaries of AI autonomy should be set.
